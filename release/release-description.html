Asset Filter v1.1.0
SDStudio 감정 이미지 자동 필터링 도구

NovelAI로 캐릭터 감정 에셋을 대량 생성하면 필연적으로 선별 작업이 뒤따릅니다.
수백 장 중에서 해부학적 결함이 없고, 감정 표현이 정확하며, 구도까지 괜찮은 이미지를 골라내는 건 상당한 시간이 드는 일입니다.

이 도구는 4종의 AI 모델을 조합하여 그 과정을 자동화합니다.
감정별로 이미지를 폴더에 넣고 실행하면, 각 감정의 상위 N장을 선별하여 출력 폴더로 복사합니다.


사용 모델

모델 역할 특징
Camie Tagger v2 감정 분류 + 결함 감지 7만개 Danbooru 태그, ONNX, DirectML GPU 가속
Aesthetic Predictor V2.5 시각적 품질 평가 SigLIP 비전 트랜스포머, 1~10점, 일러스트 특화
MediaPipe Face Detection 얼굴 프레이밍 평가 얼굴 크기·위치·신뢰도 종합, 선택적 ON/OFF
DINOv2 (facebook/dinov2-base) 참조 이미지 일관성 평가 Self-supervised 비전 트랜스포머, cosine similarity 기반

해부학 결함 자동 감점 — bad_anatomy, extra_fingers, mutated_hands 등 22개 결함 태그 중 가장 높은 확률값(max)을 기준으로 감점합니다. 결함 태그가 하나라도 높은 확률로 감지되면 점수가 크게 차감됩니다. 감점 결과는 report.json의 neg_score 필드에 기록됩니다.
EXIF 태그 결합 — NovelAI가 이미지에 기록하는 char_captions 메타데이터를 스코어링에 함께 활용합니다.
Tag Deviation Auto-Exclude — 참조 이미지에 없는 외형 특징(안경, 귀걸이, 머리 색 변경 등)이 후보 이미지에서 감지되면 자동 제외합니다.


파일명 규칙

감정별 자동 분류를 위해 아래 형식의 파일명이 필요합니다:

{캐릭터}.{감정 키워드}.{번호}.png

gabriel.acting coy.14.png
gabriel.happy smile.3.png
gabriel.angry.30.png
gabriel.crying.7.png

감정 키워드에 공백을 포함할 수 있습니다. 모든 이미지를 하나의 폴더에 넣으면 됩니다.


사용 방법

1 ZIP 파일을 다운로드하고 압축을 해제합니다.
2 AssetFilter.exe를 실행합니다. (또는 run.bat)
3 Input Folder에서 이미지 폴더를 선택하면 감정 수와 이미지 수가 자동으로 표시됩니다.
4 Top N을 설정합니다. 감정별 상위 몇 장을 선별할지 지정합니다. (기본값 10)
5 필요에 따라 Scoring Options를 조절합니다. (아래 참고)
6 Reference Consistency를 사용하려면 참조 이미지 1~5장을 추가합니다.
7 ▶ Run Filter를 클릭하면 프로그레스 바와 함께 분석이 진행됩니다.
8 완료 후 Open Output으로 결과 폴더를 확인합니다.

출력 폴더는 입력 폴더 옆에 {폴더명}_filtered 이름으로 자동 생성됩니다.


각 스코어러는 무엇을 평가하나요?

🎭 Emotion Score — "이 이미지가 정말 해당 감정을 표현하고 있는가?"

파일명에 적힌 감정 키워드(예: angry, crying)가 실제 이미지에서 얼마나 감지되는지 측정합니다. Camie Tagger가 7만개 태그 중 해당 감정 태그의 확률을 반환하며, 이것이 기본 정렬 기준입니다. NovelAI EXIF에 캐릭터 태그가 있으면 함께 사용됩니다.

→ 항상 활성. 끌 수 없음. 이것만으로도 기본적인 감정 필터링이 가능합니다.

🎨 Aesthetic Score — "이 이미지가 그림으로서 보기 좋은가?"

감정은 맞지만 구도가 어색하거나, 색감이 탁하거나, 전체적인 완성도가 떨어지는 이미지를 걸러내기 위한 점수입니다. SigLIP 비전 트랜스포머가 1~10점 스케일로 평가하며, 일러스트/애니메 스타일에 특화되어 있습니다.

→ 선택적 활성화. Quality Floor을 설정하면 기준 미달 이미지를 전역 제거할 수 있습니다.

👤 Face Framing — "표정이 잘 보이는가?"

감정 에셋에서 얼굴이 너무 작거나, 옆모습이거나, 가려져 있으면 감정 표현 확인이 어렵습니다. MediaPipe가 얼굴의 크기, 위치, 감지 신뢰도를 종합하여 "표정을 볼 수 있는 정도"를 0~1점으로 평가합니다.

→ 선택적 활성화. Hard Filter 모드면 얼굴이 안 보이는 이미지를 완전 제거. Weighted 모드면 점수에만 반영.

🔗 Reference Consistency — "참조 이미지와 같은 캐릭터처럼 보이는가?"

AI 이미지 생성 시 동일 프롬프트로도 의상 색상이 바뀌거나, 액세서리가 추가/삭제되거나, 체형이 달라지는 경우가 흔합니다. DINOv2가 참조 이미지와의 전체적인 시각 유사도를 cosine similarity로 측정합니다. 추가로 Camie Tagger 기반 Tag Deviation 필터가 안경, 귀걸이 등 외형 특징의 불일치를 자동 감지합니다.

→ 선택적 활성화. 참조 이미지 1~5장 필요. 같은 입력 폴더의 이미지나 이전 실행 결과를 참조로 사용 가능.


Scoring Options 상세

▸ Enable Aesthetic Score (기본: ON)
SigLIP 비전 트랜스포머가 이미지의 시각적 품질을 1~10점 스케일로 평가합니다.
구도, 색감, 선 품질 등을 종합적으로 판단하며 일러스트 및 애니메 스타일에 특히 높은 정확도를 보입니다.

Min Quality (Quality Floor) — 이 값 미만의 이미지는 모든 감정 그룹에서 전역 제거됩니다.
1.0필터링 없음 (모든 이미지 통과)
3.0 (기본)명백한 저품질만 제거. 대부분의 AI 생성 이미지 통과
4.0~5.0중간 품질 이상만 허용. 후보 수가 충분할 때 사용
6.0+매우 엄격. 감정별 이미지가 부족해질 수 있음

NovelAI 이미지는 대체로 4.0~7.0 범위에 분포합니다. 3.0이면 거의 탈락이 없고, 5.0이면 약 30~40%가 제거됩니다.


run.bat 실행: NVIDIA GPU가 있으면 CUDA 추론(이미지당 약 0.05초), 없으면 CPU 추론(이미지당 약 1~2초)으로 자동 전환됩니다.
▸ Enable Face Framing (기본: OFF)
MediaPipe 얼굴 감지를 통해 얼굴의 가시성과 구도를 평가합니다.
얼굴 크기 비율, 중앙 정렬도, 감지 신뢰도를 종합하여 0~1점을 산출합니다.

두 가지 모드를 제공합니다:

Hard Filter 얼굴 점수가 Threshold 미만인 이미지를 완전 제외합니다. combined score에 포함되지 않습니다.
→ 웃는 표정, 우는 표정 등 얼굴이 핵심인 감정에 적합합니다.

Weighted 얼굴 점수를 가중치로 반영하되, 이미지를 제외하지는 않습니다.
→ stretching, cozy 등 포즈가 중심인 감정에서 얼굴이 가려져도 불이익을 최소화합니다.

Threshold 가이드 (Hard Filter 모드):
0.10~0.20매우 느슨. 얼굴이 아주 작거나 옆모습이라도 통과
0.30 (기본)적당함. 정면~3/4 각도에서 얼굴이 인식 가능하면 통과
0.50+엄격. 정면 근접 얼굴만 통과. 전신/뒷모습 대부분 탈락

감정 표현 평가가 목적이면 0.25~0.35가 적합합니다. 전신 포즈가 많은 데이터셋이면 0.15~0.25로 낮추세요.

▸ Enable Reference Consistency (기본: OFF) — v1.1.0 신규
DINOv2 비전 트랜스포머로 참조 이미지 1~5장과의 시각적 유사도를 평가합니다.
의상, 체형, 색감, 전체적인 스타일이 참조와 얼마나 일치하는지를 cosine similarity로 측정합니다.

두 가지 모드를 제공합니다:

Hard Filter 일관성 점수가 Threshold 미만인 이미지를 완전 제외합니다. combined score에 포함되지 않습니다.
→ 참조와 확실히 다른 캐릭터(의상 색상 변경, 전혀 다른 디자인)를 걸러내고 싶을 때 적합합니다.
→ 간단하고 직관적: "다른 건 빼고, 나머지에서 감정+미학으로 순위 매기기."

Weighted 유사도 점수를 가중치로 반영합니다. 추가로 gate penalty가 적용됩니다.
→ 디자인 변형도 허용하되, 참조에 가까울수록 유리하게 만들고 싶을 때 적합합니다.
→ gate threshold 미만의 이미지는 추가 감점(power=3.0)을 받아 사실상 탈락합니다.

Threshold 가이드 (Hard Filter 모드):
0.30~0.40느슨. 확실히 다른 캐릭터만 제거
0.50~0.60 (기본)적당함. 의상/디자인이 비슷한 이미지만 통과
0.70+엄격. 참조와 매우 유사한 이미지만 통과. 포즈 변화도 감점됨

⚠️ Consistency 점수는 percentile 기반 정규화를 사용합니다. 절대값이 아닌 "이 데이터셋 내에서 참조에 가까운 상위 몇 %인가"로 해석하세요.


Tag Deviation Auto-Exclude (자동)
Reference Consistency를 활성화하면 자동으로 작동합니다.
참조 이미지에 없는 외형 특징(안경, 귀걸이, 머리 색 변경, 동물 귀, 뿔 등)이 후보 이미지에서 감지되면 자동 제외합니다.
88개 외형 태그(헤어스타일, 액세서리, 비인간 특징 등) 대상. 별도 설정 없이 참조 이미지만 잘 선택하면 됩니다.
▸ Weights (가중치 조합)
Emotion / Aesthetic / Face / Consistency 네 항목의 가중치를 직접 설정할 수 있습니다.
합계가 1.0이 되도록 자동 조정되며, 하나의 값을 변경하면 나머지가 비례하여 조절됩니다.
2개 이상의 스코어러가 활성화되어야 가중치 UI가 나타납니다.

Hard Filter 모드의 스코어러(Face/Consistency)는 가중치에 포함되지 않습니다.
해당 스코어러는 합격/불합격 필터로만 동작하고, 나머지 스코어러의 가중치만으로 combined score를 계산합니다.

▸ Exclude Tags (수동 제외 태그)
쉼표로 구분된 태그를 입력하면 해당 태그가 감지된 이미지를 제거합니다.
예: glasses, red_eyes, multiple_girls

Camie Tagger가 해당 태그를 0.5 이상 확률로 감지해야 제거됩니다.
EXIF 메타데이터에 해당 태그가 포함된 경우에는 의도된 것으로 간주하여 제거하지 않습니다.

Q&A — 자주 묻는 질문

Q. Top N = 1로 설정하면 어떻게 되나요?
각 감정에서 combined score가 가장 높은 이미지 딱 1장만 선택됩니다.

동작 순서:
1. 모든 필터(Quality Floor, Face Hard Filter, Consistency Hard Filter, Exclude Tags, Tag Deviation)가 먼저 적용됩니다.
2. 필터를 통과한 이미지들 중 combined score 기준으로 정렬합니다.
3. 1등 이미지만 출력 폴더에 복사됩니다.

만약 필터가 해당 감정의 모든 이미지를 제거했다면?
Hard Filter(Face 또는 Consistency)에 의해 모든 후보가 탈락한 경우, fallback으로 탈락한 이미지 중 가장 점수 높은 1장을 강제 보존합니다. report.json에 hard_filter_fallback_used: true가 표시됩니다.

주의: Top N = 1은 각 감정의 "최고점 1장"만 남기므로, 비슷한 점수의 좋은 이미지들이 모두 탈락할 수 있습니다. 처음에는 Top N = 3~5로 시작한 뒤 결과를 보고 좁히는 것을 추천합니다.
Q. 레퍼런스 이미지는 어떻게 선택해야 하나요?
좋은 레퍼런스의 기준:

• 원하는 캐릭터 디자인이 가장 정확하게 나온 이미지를 선택하세요.
• 의상, 머리 색, 체형, 액세서리가 "이것이 기준"이라고 할 수 있는 이미지여야 합니다.
• 1~5장 선택 가능하며, 여러 장을 선택하면 평균 임베딩이 기준이 됩니다.

좋은 레퍼런스 예시:
• 같은 입력 폴더에서 "이건 잘 나왔다"라고 생각하는 이미지 2~3장
• 이전 실행에서 필터링된 결과 중 디자인이 정확한 이미지
• 다른 폴더의 공식 캐릭터 시트나 원본 디자인 이미지

피해야 할 레퍼런스:
• 너무 특수한 포즈나 앵글의 이미지 (예: 극단적인 클로즈업, 뒷모습만)
• 디자인이 서로 다른 이미지를 섞어서 선택 (평균이 흐려짐)
• 해상도가 매우 낮거나 크롭이 심한 이미지

핵심 팁:
레퍼런스를 여러 장 넣으면 DINOv2가 평균 임베딩을 계산합니다. 따라서 레퍼런스끼리 디자인이 일관되어야 기준이 명확해집니다. 레퍼런스 간 디자인이 다르면 "중간"의 모호한 기준이 만들어지고, 아무 이미지도 높은 점수를 받지 못할 수 있습니다.

레퍼런스 이미지의 출처:
같은 입력 폴더의 이미지를 레퍼런스로 사용해도 됩니다. 해당 이미지는 후보에서 제외되지 않으며 동일한 스코어링 과정을 거칩니다. 다만 자기 자신과의 유사도가 당연히 높으므로 consistency 점수에서 유리합니다.
Q. 레퍼런스 이미지 간에 소품/액세서리가 다르면 어떻게 되나요?
두 가지 시스템이 독립적으로 동작합니다:

1. DINOv2 Visual Consistency (전체적 유사도)
레퍼런스 이미지의 임베딩을 평균냅니다. 레퍼런스 A가 "안경 있음", 레퍼런스 B가 "안경 없음"이면 평균 임베딩은 "어중간한 상태"가 됩니다. 이 경우:
• 안경이 있든 없든 둘 다 중간 정도의 유사도를 받게 됩니다.
• 명확한 기준이 없어지므로 threshold를 낮추지 않으면 좋은 이미지가 걸러질 수 있습니다.

2. Tag Deviation Auto-Exclude (외형 태그 기반)
레퍼런스 이미지의 태그 확률도 평균냅니다. 예를 들어:
레퍼런스 A: glasses 확률 0.95레퍼런스 B: glasses 확률 0.05평균: 0.50

후보 이미지의 glasses 확률이 0.90이면, 편차 = 0.90 - 0.50 = 0.40.
편차 threshold(0.30) 이상이므로 제외될 수 있습니다.

즉, 레퍼런스 간에 소품이 불일치하면 해당 소품은 "절반 정도 있는" 상태로 간주되어, 소품이 확실히 있는 이미지와 확실히 없는 이미지 모두가 편차 제외 대상이 될 수 있습니다.

권장: 레퍼런스 이미지는 소품/액세서리가 동일한 것으로 통일하세요. 디자인에 변형이 있으면 가장 자주 나타나는 버전을 기준으로 선택하세요. 또는 Consistency의 threshold를 낮추어 완화하세요.
Q. 레퍼런스 이미지를 몇 장 넣는 게 좋나요?
2~3장을 추천합니다.

• 1장: 해당 이미지의 특정 포즈/구도에 과적합될 수 있습니다. 같은 캐릭터지만 다른 포즈면 낮은 점수를 받을 수 있습니다.
• 2~3장: 포즈 변화를 평균에 흡수하면서 캐릭터 디자인의 핵심(의상, 색감)에 집중할 수 있습니다.
• 4~5장: 평균이 더 안정적이지만, 이미 충분한 경우가 많습니다. 모든 레퍼런스가 동일 디자인이라면 효과적입니다.

핵심은 양이 아니라 일관성입니다. 디자인이 동일한 이미지 2장이 디자인이 다른 이미지 5장보다 훨씬 좋은 기준을 만듭니다.
Q. "acting coy"처럼 Camie Tagger에 없는 감정은 어떻게 되나요?
Camie Tagger의 7만개 태그 어휘에 해당 감정 키워드가 없으면, 모든 이미지의 emotion score가 0.0으로 산출됩니다.

이 경우 자동 fallback이 작동합니다:
• Emotion 가중치를 0으로 만들고, 활성화된 보조 스코어러(Aesthetic, Face, Consistency)의 가중치를 재분배하여 합계 1.0으로 만듭니다.
• 예: Emotion 0.65 + Aesthetic 0.35 → Emotion 0.0 + Aesthetic 1.0
• 예: Emotion 0.45 + Aesthetic 0.25 + Consistency 0.20 → Aesthetic 0.56 + Consistency 0.44

결과적으로 "감정은 몰라도 품질/일관성이 좋은 이미지"가 우선 선택됩니다. report.json에 fallback_auxiliary_only: true가 표시됩니다.

Camie Tagger는 감정 키워드의 개별 단어도 시도합니다. "happy smile"이면 "happy"와 "smile" 각각을 찾아 평균낸 결과를 사용합니다.
Q. Hard Filter와 Weighted 중 어떤 모드를 써야 하나요?
Hard Filter를 추천하는 경우:
• 기준 미달 이미지가 절대 포함되면 안 되는 경우
• 이분법적인 판단이 필요한 경우 ("얼굴이 보이냐 안 보이냐", "같은 캐릭터냐 아니냐")
• 설정이 간단하고 결과를 예측하기 쉬움

Weighted를 추천하는 경우:
• 기준 미달이더라도 다른 요소가 뛰어나면 허용하고 싶은 경우
• 점수 분포가 연속적이어서 이분법이 부적절한 경우
• 여러 요소를 종합적으로 반영하고 싶은 경우

실전 팁: 처음에는 Hard Filter로 시작하세요. 결과를 보고 "괜찮은 이미지가 너무 많이 잘렸다"면 threshold를 낮추거나 Weighted로 전환하세요. Weighted는 가중치 조절이 추가되므로 조금 더 복잡합니다.
Q. 이 도구의 인식 한계점은 무엇인가요?
1. Camie Tagger의 감정 인식 한계
• 어휘 제한: Camie Tagger는 Danbooru 태그 기반입니다. "acting coy", "admiring", "pouting" 등 세부 감정은 어휘에 없어서 emotion score가 0.0이 됩니다. 이 경우 보조 스코어러 fallback에 의존합니다.
• 복합 감정 해석: "happy smile"은 "happy"와 "smile"을 개별로 찾아 평균내지만, 두 태그의 뉘앙스 차이(기쁜 미소 vs 억지 미소)는 구분하지 못합니다.
• 포즈 기반 감정: "stretching", "exhausted" 같은 신체 동작 기반 감정은 태그 매칭이 어렵거나 부정확할 수 있습니다. 이런 감정은 Aesthetic + Face 기반으로 결과 품질이 좌우됩니다.

2. Face Detection의 한계
• 2D 일러스트 한계: MediaPipe는 실사 사진에 최적화되어 있습니다. 극단적으로 양식화된 애니메 얼굴(눈이 매우 크거나, 코가 없거나)에서는 감지 실패율이 높아집니다.
• 옆모습/뒷모습: 3/4 이상 돌아간 얼굴이나 뒷모습은 감지되지 않아 NO_FACE_SCORE(0.3)를 받습니다.
• 가려진 얼굴: 손으로 얼굴을 가리거나 머리카락으로 눈이 가려진 경우 감지 실패 또는 낮은 신뢰도로 저점수를 받습니다. 이는 "crying" 감정에서 손으로 눈물을 닦는 포즈에 불이익이 될 수 있습니다.

3. DINOv2 Consistency의 한계
• 포즈 vs 디자인 구분 불가: DINOv2는 전체적인 시각 유사도를 측정합니다. 같은 캐릭터가 다른 포즈를 취하면 유사도가 떨어지고, 다른 캐릭터가 같은 포즈를 취하면 유사도가 올라갑니다. 즉, "포즈 유사도"와 "디자인 유사도"를 분리하지 못합니다.
• 배경 영향: 배경이 크게 달라지면 (밝은 배경 → 어두운 배경) 같은 캐릭터라도 유사도가 낮게 나올 수 있습니다. DINOv2는 캐릭터만이 아니라 이미지 전체를 인코딩합니다.
• 색상 변이: 의상의 색상이 미묘하게 다른 경우(연보라 → 진보라), DINOv2는 이를 감지하지만 "같은 디자인의 색상 변이"인지 "다른 디자인"인지 판단하지 못합니다.
• 상대 점수: Percentile 기반 정규화이므로, 모든 이미지가 참조와 비슷하면 점수 차이가 작아지고 노이즈에 민감해집니다. 반대로 확실히 다른 이미지가 많으면 구분이 잘 됩니다.

4. Tag Deviation의 한계
• 88개 태그만 검사: 외형 태그 목록은 수동으로 정의되어 있습니다. 목록에 없는 액세서리나 특징은 감지되지 않습니다.
• 확률 기반: Camie Tagger의 태그 확률에 의존하므로, 모델이 태그를 잘못 예측하면 정상 이미지가 제외되거나 비정상 이미지가 통과할 수 있습니다.
• 색상 변이 미감지: Tag Deviation은 태그의 유무만 확인합니다. "black_hair" → "dark_brown_hair" 수준의 미묘한 색상 차이는 두 태그가 모두 감지되거나 모두 감지되지 않아서 걸러지지 않을 수 있습니다.

5. Aesthetic Score의 한계
• 주관적 품질: SigLIP 기반 모델의 "미학" 판단은 학습 데이터의 평균적 선호도를 반영합니다. 의도적으로 거친 스타일이나 독특한 색감은 낮은 점수를 받을 수 있습니다.
• 해상도 영향: 같은 이미지라도 해상도나 압축 품질이 달라지면 점수가 변합니다.

6. 일반적 한계
• 감정별 독립 평가: 각 감정은 독립적으로 평가됩니다. "angry 1등"과 "sad 1등"이 같은 이미지가 될 수 있습니다. 감정 간 중복 제거 기능은 없습니다.
• PNG만 지원: 현재 .png 파일만 인식합니다.
• 단일 캐릭터 가정: 파일명 형식이 단일 캐릭터를 가정합니다. 여러 캐릭터가 포함된 이미지(예: 2인 구도)는 별도 처리가 없습니다.
Q. report.json에는 무엇이 포함되나요?
출력 폴더에 생성되는 report.json은 전체 스코어링 결과를 담고 있습니다:

• config: 사용된 모든 설정값 (가중치, threshold, 모델명, 참조 이미지 경로 등)
• summary: 전체 이미지 수, 발견된 감정 수, 선택된 이미지 수
• emotions: 감정별로 모든 이미지의 점수 상세:
  - emotion_score, aesthetic_score, face_score, consistency_score, combined_score
  - neg_score (해부학 결함 감점)
  - consistency_raw_score (정규화 전 raw cosine similarity)
  - filtered_by_face, filtered_by_consistency, penalized_by_consistency_gate 등 필터링 메타데이터

결과가 예상과 다를 때 report.json을 확인하면 어떤 이미지가 왜 탈락/선택되었는지 진단할 수 있습니다.
Q. 감정이 아니라 동작/포즈 에셋을 선별할 때 권장 설정은?
동작/포즈 에셋은 전신 포즈 변화가 크기 때문에 감정 에셋과 설정을 다르게 해야 합니다.

핵심 차이점:
• 얼굴이 작거나 안 보이는 경우가 많으므로 Face는 OFF 권장
• 포즈 변화가 크면 DINOv2 유사도가 자연스럽게 낮아지므로 Consistency Threshold를 0.40으로 낮춤
• Tag Deviation이 포즈에 따른 옷/머리 인식 변화에 과민 반응할 수 있음

레퍼런스 미사용:
AestheticON, Quality Floor 3.0
FaceOFF
ConsistencyOFF
WeightsEmotion 0.65 / Aesthetic 0.35 (기본값)

레퍼런스 사용 (캐릭터 유지 + 동작 선별):
AestheticON, Quality Floor 3.0
FaceOFF
ConsistencyWeighted (포즈 변화에 의한 유사도 하락을 부드럽게 반영)
WeightsEmotion 0.45 / Aesthetic 0.25 / Consistency 0.30
레퍼런스 팁중립 서있기 등 기본 포즈 2~3장 사용. 역동적 포즈를 레퍼런스로 넣으면 해당 포즈에 과적합됨

⚠️ 주의: 현재 레퍼런스는 단일 세트이므로, 여러 캐릭터를 한 번에 돌리면 모든 캐릭터가 하나의 기준에 비교됩니다. 다캐릭터 동작 선별 시에는 캐릭터별로 폴더를 분리하여 각각 실행하세요.
